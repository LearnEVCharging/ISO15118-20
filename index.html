About ISO15118-20 Protocol

Protocol Overview

ISO15118-20 represents a significant evolution in electric vehicle charging communication standards. Building upon its predecessors, this latest version introduces revolutionary capabilities that transform the EV charging experience.

Key Improvements

  • Enhanced Plug & Charge: Streamlined authentication and authorization process with multiple contract support per session
  • Bidirectional Power Transfer: Native support for Vehicle-to-Grid (V2G), Vehicle-to-Home (V2H), and Vehicle-to-Load (V2L)
  • Advanced Security: Mutual TLS 1.3 encryption with larger certificate key sizes (secp521r1/Ed448)
  • Wireless Power Transfer: Support for wireless communication and automated connection devices
  • Improved Interoperability: Better compatibility across different manufacturers and regions

Benefits for Stakeholders

  • Seamless charging experience with automatic authentication
  • Secure payment processing without physical cards
  • Energy cost optimization through smart charging
  • Revenue opportunities through V2G services

  • Reduced operational costs and complexity
  • Improved customer satisfaction and loyalty

For Grid Operators

  • Peak load management and demand response

  • Distributed energy storage capabilities

Implementation Timeline

The ISO15118-20 standard is being progressively adopted across the industry, with major OEMs and charging infrastructure providers implementing support throughout 2025 and 2026. Early adopters are already seeing significant benefits in terms of user experience and operational efficiency.

Technical Specifications

Communication Architecture

ISO15118-20 implements a sophisticated communication stack that enables secure, efficient data exchange between EVs and charging infrastructure using advanced protocols and security measures.

Security Framework +

Mutual TLS 1.3 Implementation

  • Enhanced encryption algorithms with Perfect Forward Secrecy
  • Mutual authentication between vehicle and charging station
  • Reduced handshake latency for faster connection establishment
  • Larger key sizes (secp521r1/Ed448) for stronger classical security — post-quantum algorithms remain a future work item at ISO/CharIN

Certificate Management

  • PKI-based authentication with secp521r1 or Ed448 curves
  • Automatic certificate provisioning and renewal
  • Enhanced OCSP certificate validation
  • Support for multiple contract certificates per session

Power Transfer Protocols +

Bidirectional Charging Support

  • AC and DC bidirectional power flow capabilities
  • Grid-to-Vehicle (G2V) charging optimization
  • Vehicle-to-Grid (V2G) energy injection
  • Vehicle-to-Home (V2H) and Vehicle-to-Load (V2L) support

Smart Charging Features

  • Dynamic load balancing across multiple charging points
  • Time-of-use optimization and demand response
  • Grid constraint management and congestion relief
  • Renewable energy integration and carbon footprint reduction

Message Formats and Processing +

EXI Encoding Enhancement

  • Efficient XML Interchange (EXI) with improved compression
  • Reduced message size for faster transmission
  • Enhanced parsing capabilities and schema validation
  • Support for new message types and parameters

Message Categories

  • Session establishment and management messages
  • Authentication and authorization protocols
  • Charging parameter negotiation and optimization
  • Power delivery control and monitoring
  • Bidirectional power flow coordination

Implementation Requirements

Successfully implementing ISO15118-20 requires careful attention to both hardware and software components:

  • Hardware Compatibility: Bidirectional power electronics and enhanced security modules
  • Software Stack: TLS 1.3 support with updated cryptographic libraries
  • Certificate Infrastructure: PKI integration with automatic certificate management
  • Testing and Validation: Comprehensive conformance and interoperability testing
  • Security Audits: Regular security assessments and vulnerability testing

Frequently Asked Questions

Find answers to common questions about ISO15118-20 implementation and capabilities

What is ISO15118-20 and how does it differ from ISO15118-2?

ISO15118-20 is the latest evolution of the EV charging communication standard, introducing bidirectional power transfer (V2G), enhanced security with mutual TLS 1.3, wireless power transfer support, and improved plug & charge functionality. Key differences include larger certificate key sizes (secp521r1/Ed448 vs secp256r1), multiple contract installation per session, and support for automated connection devices.

What are the main benefits of implementing ISO15118-20?

ISO15118-20 offers enhanced security through mutual authentication, bidirectional charging capabilities for V2G applications, improved interoperability across different charging networks, wireless communication support, and streamlined plug & charge experiences. It also enables advanced use cases like smart charging optimization and energy grid stabilization.

Is ISO15118-20 backward compatible with ISO15118-2?

ISO15118-20 is designed with migration pathways from ISO15118-2, but is not directly backward compatible due to changes in certificate key sizes, TLS requirements, and message formats. However, systems can be designed to support both standards during transition periods.

What is the current adoption status of ISO15118-20?

ISO15118-20 is in the early adoption phase with major OEMs and charging infrastructure providers beginning implementations. The standard was published in 2022, and we're seeing increasing interest from automotive manufacturers, charging network operators, and certification bodies.

What are the key technical requirements for ISO15118-20 implementation?

Key requirements include support for secp521r1 or Ed448 cryptographic curves, mutual TLS 1.3 implementation, updated PKI infrastructure, new EXI codec for message processing, and enhanced OCSP certificate validation. Hardware Security Modules (HSMs) may need upgrades to support the new key formats.

How does bidirectional charging benefit EV owners?

Bidirectional charging allows EV owners to earn revenue by selling energy back to the grid during peak demand periods, reduce electricity costs through smart charging optimization, and provide backup power for their homes during outages. This transforms EVs from purely cost centers to revenue-generating assets.

What testing is required for ISO15118-20 compliance?

Compliance testing includes protocol conformance testing, interoperability testing with multiple vendor implementations, security validation including penetration testing, and performance testing under various load conditions. Certification bodies provide standardized test procedures and validation services.

How does ISO15118-20 support wireless power transfer?

ISO15118-20 includes provisions for wireless communication during wireless power transfer, enabling the same secure authentication and authorization processes used in wired charging. This supports the development of automated parking and charging systems for future autonomous vehicles.
